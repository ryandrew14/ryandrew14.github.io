# 2026-personal-site

My personal website — a static site generated by a small Rust program.

## How it works

- Content lives in [`content/`](content) as Markdown files with TOML front matter
  (`+++ ... +++`).
- [`src/main.rs`](src/main.rs) parses each Markdown file with
  [`pulldown-cmark`](https://crates.io/crates/pulldown-cmark) and renders it
  into HTML using [`tera`](https://crates.io/crates/tera) templates from
  [`templates/`](templates).
- Static assets (CSS, images) live in [`static/`](static) and are copied as-is.
- Everything is written to `dist/`, a plain static site ready to deploy
  anywhere (GitHub Pages, Netlify, S3, ...).

## Building

```sh
cargo run
```

This generates the site into `dist/`. To build and preview it locally in one
step, pass `serve`:

```sh
cargo run -- serve
```

This rebuilds the site and serves it at <http://localhost:8000> (Ctrl-C to
stop). Re-run the command to pick up content changes.

### Deploying under a sub-path

If the site is hosted at a sub-path (e.g. a GitHub Pages project site at
`https://<user>.github.io/<repo>/`), set `SITE_BASE_PATH` so links and assets
resolve correctly:

```sh
SITE_BASE_PATH=/2026-personal-site/ cargo run --release
```

## Adding content

- **About / home page**: edit [`content/about.md`](content/about.md).
- **Links page**: edit [`content/links.md`](content/links.md).
- **Blog posts**: add a new Markdown file to [`content/blog/`](content/blog).
  Each post needs front matter like:

  ```
  +++
  title = "My Post Title"
  date = "2026-06-12"
  description = "A short summary."
  +++

  Post content goes here...
  ```

  The filename (without `.md`) becomes the post's URL slug.

## Deployment

[`.github/workflows/deploy.yml`](.github/workflows/deploy.yml) builds the
site and publishes `dist/` to GitHub Pages on every push to `main`. Enable
GitHub Pages for this repo with the source set to **GitHub Actions**
(Settings → Pages → Build and deployment → Source).
